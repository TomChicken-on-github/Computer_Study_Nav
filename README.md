### 如何高效的学习/解决问题（重要！） 

> 相同工具在不同的人手中能够被创造出来的价值不同

- 首先，务必培养擅用搜索引擎/工具的能力（重要），对网络上的任何资料都要有自己的主观意识，~~批判性欣赏（x）~~ 批判性思维去看待，多查阅Wiki、文档。
- 学习过程中，要多上手实践，有助于你对概念、理论知识的快速吸收 ~~飞升成w6大手子~~ ，练习你动手解决问题的能力，但凡原因用心实践基本就不会忘记对应的知识啦，所以其实计算机不难学捏
- 遇到问题后可以先尝试独立思考排错解决，若该问题有对应的错误日志，请尝试分析它，通常能够直接得知问题出现的原因；若无法独立解决，查阅相关文档/Wiki/自述文件（如READNE.md），利用搜素引擎；实在不行再去社交平台求助他人！求助他人前必须要认清一个现实：**没有人有义务为你提供无偿帮助** ，所以，提问时请带上脑子，详细描述存在的问题（包括但不限于：问题是什么、问题是如何出现的、如何复现该问题、问题出现时所处的环境、问题出现时伴随的日志文件等），只有问题问的足够清晰明了别人才能明白问题出在哪才能够协助你，否则神仙来了也不一定能帮的了你（tips:算命先生算命也要看你的手相面相才能算捏ouo）最后，务必**端正态度**友善交流，因为**没有人有义务为你提供无偿帮助**，否则大概率不会有佬帮你，搞不好还要挨骂zwz
- 解决问题后，可以将问题详情（与上文详细的问题描述方法相同的方法）、问题的原因发现过程、排错过程、日志分析过程、思考过程、解决过程图文并茂记录下来 最后总结引发问题的根本原因 对其整理成文档（优先使用Markdown文档，简单易上手功能强大）后归档，不要嫌麻烦，将来发生相同（或类似）问题时你一定会大吃一惊的，能查阅你自己整理的文档快速解决问题，节省大量时间同时满满成就感（有能力可以自建博客，更方便查阅也一目了然，没事的时候多看几眼加深印象，说不定还能同时帮助到了别人捏）
